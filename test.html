### Practical 1: Draw and explain architecture of the web browser. List and explain various HTML request and response headers. #### Architecture of the Web Browser A web browser is a software application that retrieves, presents, and traverses information resources on the World Wide Web. Its architecture typically consists of the following components: 1. User Interface (UI) : The front-end layer where users interact, including the address bar, back/forward buttons, bookmarks, and the rendering area for web pages. 2. Browser Engine : Acts as an intermediary between the UI and the rendering engine, handling actions like page loading, reloading, and navigation. 3. Rendering Engine : Responsible for parsing HTML/CSS and displaying the content. Examples include Blink (used in Chrome), Gecko (Firefox), and WebKit (Safari). It interprets HTML to build the DOM (Document Object Model), CSS to style it via CSSOM, and combines them into a render tree for painting on the screen. 4. Networking : Handles HTTP/HTTPS requests, DNS resolution, and data transfer between the browser and servers. 5. JavaScript Interpreter : Executes JavaScript code, often using engines like V8 (Chrome) or SpiderMonkey (Firefox). 6. UI Backend : Draws basic widgets like buttons and windows using OS-specific APIs. 7. Data Storage : Manages cookies, local storage, session storage, and cache for persistent data. A simplified "drawing" (text-based diagram) of the architecture: ``` +-------------------+ | User Interface | +-------------------+ | v +-------------------+ | Browser Engine | +-------------------+ | +----------------> +-------------------+ | | Networking | | +-------------------+ v ^ +-------------------+ | | Rendering Engine | <----------------+ +-------------------+ | +----------------> +-------------------+ | | JavaScript Engine | | +-------------------+ v +-------------------+ | UI Backend | +-------------------+ | v +-------------------+ | Data Storage | +-------------------+ ``` #### HTTP Request and Response Headers HTTP headers are key-value pairs sent between client (browser) and server to provide metadata about the request/response. Common Request Headers: - Host : Specifies the domain name of the server (e.g., `Host: example.com`). - User-Agent : Identifies the browser and OS (e.g., `User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)`). - Accept : Indicates acceptable content types (e.g., `Accept: text/html, application/xhtml+xml`). - Accept-Language : Preferred languages (e.g., `Accept-Language: en-US,en;q=0.9`). - Authorization : Credentials for authentication (e.g., `Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l`). - Cookie : Sends stored cookies to the server (e.g., `Cookie: sessionid=abc123`). - Referer : URL of the page making the request (e.g., `Referer: https://example.com/page`). Common Response Headers: - Content-Type : Specifies the MIME type of the response body (e.g., `Content-Type: text/html; charset=UTF-8`). - Content-Length : Size of the response body in bytes (e.g., `Content-Length: 348`). - Set-Cookie : Instructs the browser to store cookies (e.g., `Set-Cookie: sessionid=abc123; Max-Age=3600`). - Location : Used in redirects (e.g., `Location: https://example.com/newpage`). - Server : Identifies the server software (e.g., `Server: Apache/2.4.41`). - Cache-Control : Directives for caching (e.g., `Cache-Control: max-age=3600`). - ETag : A validator tag for caching (e.g., `ETag: "33a64df551425fcc55e4d42a148795d9f25f89d4"`). ### Practical 2 #### a.
